WDD 131 Final Project Site Plan - George Olwal

Site Name

CropDaktari

Daktari is a Swahili word for doctor, therefore CropDaktari was coined to effectively communicates the main purpose of the website - diagnosis of crop diseases and pests. It's concise, memorable, and relevant to the project's goal of crop management and protection.

Site Purpose

CropDaktari serves as a comprehensive tool for farmers to:

  • Diagnose crop diseases using image recognition technology.
  • Identify pests affecting their crops.
  • Access location-based data on crop health and pest prevalence.
  • View statistics and information on crop diseases and pests.
  • Get recommendations for disease and pest management.

The site aims to promote precision farming methods, improve crop health, and contribute to food security by enabling data-driven decision-making in agriculture.

Scenarios

  1. How can I quickly identify a disease affecting my tomato plants using a photo?
  2. What are the most common pests in my area based on recent reports from other farmers?
  3. How do I access historical data on crop diseases in my region to plan for the next growing season?

Color Schema

Secondary Color
Text Color
Accent Color

The primary color (hsl(120, 60%, 50%)) represents growth and health, used for main headings and interactive elements.

The secondary color (hsl(45, 100%, 50%)) complements the primary color and is used for highlights and secondary elements.

The text color (hsl(0, 0%, 20%)) ensures good readability against light backgrounds.

The accent color (hsl(0, 100%, 50%)) is used sparingly for warnings or important information.

The background color (hsl(0, 0%, 98%)) provides a clean and neutral base for the site.

Typography

Headings: Roboto, sans-serif

Body Text: Open Sans, sans-serif

This combination provides a modern look while ensuring readability across devices.

Site Content

Home Page

Landing page for the website with introduction and brief sectional information to other pages.

Disease Diagnosis

  • Capture image and use external API to detect plant disease and general health.
  • Use external API to get map location of the captured image.
  • Use internal API to log in details of diagnosis including location.

Pest Diagnosis

  • Capture image and use external API to detect the pest.
  • Use external API to get map location of the captured image.
  • Use internal API to log in details of the pest including location.

Statistics & Information Page

  • Query and view existing diagnosis.
  • Query affected crop information and use external API to get disease and pest details and possible treatment.

Contact Page

  • Provide contact information.
  • Provide contact form.

Project Rationale

This project will help express mastery of local and external API implementations, incorporating wayfinding and other concepts learned in the WDD231 course. It's part of a larger initiative to build a crop management system focused on precision farming. The goal is to enable farmers to make data-driven decisions that can improve farm efficiency, boost production, and alleviate food insecurity. Additionally, the sparing use of resources contributes to combating the effects of climate change.
